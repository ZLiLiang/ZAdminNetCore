## 部署教程
以下是基于docker进行部署的
***

### 部署MySQL
启动MySQL容器

```
docker run \
	-d \
	--name Zmysq \
	-p 3309:3306 \
	-e MYSQL_ROOT_PASSWORD=123456 \ 
	mysql
```

<font color=Salmon>-d</font> 后台运行，<br>
<font color=Salmon>--name</font> 容器名字，<br>
<font color=Salmon>-p</font> 端口映射，<br>
<font color=Salmon>-e</font> 容器的环境配置，例如设置MySQL密码<br>

***

### 编写写Web的Dockerfile
将项目发布后，编写构建镜像需要的Dockerfile

```
FROM mcr.microsoft.com/dotnet/aspnet:7.0 AS base
WORKDIR /app
EXPOSE 8888

FROM mcr.microsoft.com/dotnet/sdk:7.0 AS build
WORKDIR /src
COPY . .	
ENTRYPOINT ["dotnet", "Z.Admin.WebApi.dll"]
```

<font color=Salmon>EXPOSE</font> 暴露端口，<br>
<font color=Salmon>WORKDIR</font> 创建工作目录，<br>
<font color=Salmon>COPY . .</font> 将当前目录下的文件拷贝到工作目录下。第一个.是当前目录下的文件，第二个.是指工作目录<br>
<font color=Salmon>ENTRYPOINT</font> 
容器启动后执行的命令,让容器执行表现的像一个可执行程序一样,与CMD的区别是不可以被 docker run 覆盖, 会把 docker run 后面的参数当作传递给，<br>

```
#构建镜像命令
docker build -t zadmin .

#运行容器命令
docker run \
	-itd \
	-p 8888:8888 \
	--name zadmincore \
	zadmin \
	/bin/bash
```

注：appsettings.json的 "urls": "http://172.17.0.6:8888", 需要修改为 0.0.0.:8888 或者 容器地址:8888

### 部署Vue项目

```
#启动nginx的容器
docker run -d -p 8081:80 --name nginx nginx

#将编译后的vue文件拷备到容器中
docker cp vue/dist/. 容器ID:/usr/share/nginx/html

#修改 /etc/nginx/conf.d/default.conf 文件
server {
        listen       80;
        server_name  localhost;

        access_log /var/log/nginx/admin_access.log;
        error_log /var/log/nginx/admin_error.log;

        location / {
                root   /usr/share/nginx/html;   
                index  index.html index.htm;
                try_files $uri $uri/ /index.html;
        }

        location /prod-api/ {
                proxy_pass http://172.17.0.6:8888/;
        }

        location /msgHub {
                proxy_pass http://172.17.0.6:8888/msgHub;
                #启用http长连接支持websocket
                proxy_http_version 1.1;
                proxy_set_header Upgrade $http_upgrade;
                proxy_set_header Connection "upgrade";
        }

        error_page   500 502 503 504  /50x.html;
}

```


完成部署
